import bpyimport mathdef clean_scene() -> None:
    """Pulisce la scena iniziale in modalità headless."""
    try:
        if bpy.context.active_object:
            bpy.ops.object.mode_set(mode='OBJECT')
        bpy.ops.object.select_all(action='SELECT')
        bpy.ops.object.delete(use_global=False)
    except Exception:
        pass # Previene crash se eseguito senza un context UI valido

Spiegazione Tecnica
Validazione Dinamica (Try/Except): La funzione setup_engine() usa il blocco try per tentare l'attivazione del raytracing tramite le API di Eevee Next (Blender 4.x). Se l'attributo manca nelle versioni 3.x, il blocco except AttributeError esegue il fallback allo Screen Space Reflection standard. Lo stesso approccio è usato in clean_scene() per evitare blocchi causati dall'assenza del Context UI in esecuzione CLI.
Isolamento e Type Hinting: L'impiego di -> tuple[bpy.types.Collection, bpy.types.Collection] e la creazione manuale delle collection isolano i dati dalla master scene. Questo prepara un ambiente ottimizzato per quando istanzieremo i Geometry Nodes su Collection_Hidden.
Costruzione Vettoriale (Metaball): Generare una complessa mesh organica per vertici da zero richiederebbe migliaia di righe di codice. La funzione spawn_xenomorph_base sfrutta la fusione volumetrica intrinseca delle Metaball. La lista coords definisce le traslazioni spaziali: estendendo i vettori lungo l'asse Y negativo e sollevandoli gradualmente su Z, calcoliamo matematicamente l'inclinazione aerodinamica del cranio.
Per trasformare questo volume base in un esoscheletro biomeccanico realistico, dobbiamo assegnargli un materiale procedurale scuro e viscido direttamente via codice.
