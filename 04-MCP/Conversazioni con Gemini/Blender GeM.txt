Ciao! L'idea di pilotare un colosso 3D come Blender tramite una gemma Ruby, e per di più sfruttare il nuovissimo Antigravity CLI per scriverla... è un progetto da veri "mad scientist" del codice. Mi piace tantissimo l'ambizione!
Prima di lanciare gli agenti a scrivere codice, facciamo un piccolo bagno di realtà (ma facilmente risolvibile) tecnico:
Blender parla nativamente Python (tramite il suo modulo bpy). Non esiste un interprete Ruby integrato. Quindi, per "programmare da zero Blender" e fare tutto quello che vuoi, la tua gemma dovrà comportarsi come un ponte (Bridge) tra i due linguaggi.
Ci sono due architetture principali che la tua gemma può usare:

Generazione ed Esecuzione Headless: La tua gemma Ruby genera dinamicamente dei file .py dietro le quinte e lancia Blender da terminale in background (usando i flag -b e -P) per eseguire il lavoro e sputare fuori il render o il file 3D.
Server RPC / WebSocket (Interattivo): Uno script Python gira dentro Blender e apre una porta locale. La tua gemma Ruby si connette a questa porta e invia comandi (in JSON) in tempo reale. Questo è molto più potente se vuoi vedere gli oggetti apparire nello schermo di Blender mentre esegui il tuo codice Ruby.
Dato che vuoi usare Antigravity CLI, possiamo sfruttare la sua capacità di editing multi-file e ragionamento multi-step per fargli scrivere sia il lato Ruby che il lato Python. Ecco come puoi orchestrarlo.

1. Inizializza il progetto
Crea l'impalcatura standard della tua gemma direttamente dal terminale:

2. Definisci le regole d'ingaggio per Antigravity
Antigravity CLI lavora al meglio quando ha un contesto chiaro a livello di progetto. Crea un file chiamato AGENTS.md nella root della tua gemma per impostare i paletti per l'agente.
AGENTS.md
L'obiettivo di questo progetto è creare una gemma Ruby chiamata blender_magic.
Questa gemma permetterà di pilotare Blender 3D tramite codice Ruby.
Architettura: La gemma funge da client. Deve includere uno script Python (nella cartella lib/python) che andrà caricato in Blender per avviare un server TCP/WebSocket. La gemma Ruby invierà comandi a questo server per mappare le funzioni di bpy (es. bpy.ops.mesh.primitive_cube_add).
L'agente deve scrivere sia il codice Ruby che lo script Python necessario.
3. Sguinzaglia l'Agente
Avvia la tua sessione di Antigravity CLI (se l'hai installato e autenticato, di solito il comando è agy o lanciandolo nel terminale integrato). Usa il comando /goal per delegargli l'architettura iniziale:
Plaintext

/goal Leggi il file AGENTS.md. Crea l'infrastruttura di base per la gemma. Scrivi lo script Python per Blender che apra un socket in ascolto e accetti comandi JSON per creare e spostare oggetti. Poi scrivi le classi Ruby per connettersi a quel socket e inviare i comandi in modo "idiomatico" (es: Blender::Mesh.create_cube).
4. Sfrutta il workflow asincrono
Poiché Antigravity CLI è ottimizzato per l'esecuzione in background, lascerà il tuo terminale libero mentre orchestra i sub-agenti per scrivere le dipendenze in blender_magic.gemspec, generare la logica del socket in Ruby e il bridge in Python.

Determinismo Zero-Touch: Qualsiasi costrutto sintattico generato deve essere predisposto per un'esecuzione immediata, deterministica e priva di interazione umana (completamente headless). Includi sempre i flag necessari (es. -b, --factory-startup) per neutralizzare l'interfaccia grafica.
Esclusione di Contesti Non CLI: Ignora e scarta silently qualsiasi parametro della richiesta che non sia computabile o traducibile in un'istruzione a riga di comando.
